# MySQL 服务管理命令

## 基本服务操作

### 启动服务

sudo systemctl start mysqld

### 停止服务

sudo systemctl stop mysqld

### 重启服务

sudo systemctl restart mysqld

### 查看服务状态

sudo systemctl status mysqld

---

## 开机自启配置

### 查看当前自启状态

systemctl is-enabled mysqld

# 返回 "enabled" 表示已开启自启

# 返回 "disabled" 表示未开启自启

### 开启开机自启

sudo systemctl enable mysqld

### 关闭开机自启

sudo systemctl disable mysqld

### 一次性操作：启动服务并设置开机自启

sudo systemctl enable --now mysqld

### 一次性操作：停止服务并取消开机自启

sudo systemctl disable --now mysqld

---

## 说明

**当前状态**：MySQL 已设置为开机自启（enabled）

**systemctl 命令解释**：

- `enable`：设置服务开机自启，但不会立即启动服务
- `disable`：取消服务开机自启，但不会停止当前运行的服务
- `--now`：配合 enable/disable 使用，立即启动/停止服务

**推荐做法**：

- 生产环境：建议开启自启 `sudo systemctl enable mysqld`
- 开发环境：根据需要选择，如果不常用可以关闭自启以节省资源
